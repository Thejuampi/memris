Java 21 In-Memory Engine

Build low-latency data paths without leaving the JVM heap.

Memris combines primitive-backed columnar storage, ByteBuddy-generated tables, and a Spring Data-style query surface to keep hot paths reflection-free and predictable.

Read the README See architecture

Runtime shape

  1. Repository method tokenized by QueryMethodLexer
  2. QueryPlanner resolves to CompiledQuery
  3. HeapRuntimeKernel dispatches by TypeCodes
  4. Generated table scans primitive page columns directly

Why Memris

Heap-first by design

100% heap-based primitive columns keep setup simple and predictable.

Zero-reflection hot paths

Build-time metadata + generated tables avoid reflective dispatch at runtime.

Familiar query surface

Use Spring Data-style method naming like findBy*, countBy*, and existsBy*.

Practical indexing

Hash indexes for O(1) lookups and range indexes for O(log n) traversals.

How it works

Build-time

Entity metadata -> generated table

Memris scans entities once, builds table metadata, and emits optimized table classes with ByteBuddy.

Runtime

Compiled query -> direct scans

The runtime kernel executes pre-resolved operations and scans typed columns with index and type-code dispatch.

  • O(1) first
  • Primitive-only APIs
  • Static TypeCodes
  • Dense arrays over maps

Query capabilities

Memris supports broad query-method operator coverage, including comparisons, string matching, null checks, ranges, ordering, and limits.

  • Comparison operators: EQ, NE, GT, GTE, LT, LTE, BETWEEN
  • String operators: LIKE, STARTING_WITH, ENDING_WITH, CONTAINING, IGNORE_CASE
  • Collection and logical operators: IN, NOT_IN, AND, OR
  • JPQL-like @Query subset for explicit query text

Concurrency guarantees

Multi-reader safe

Queries can run concurrently using lock-free lookups and safe published row visibility.

Multi-writer support

Row writes coordinate with CAS + SeqLock semantics for consistent typed reads.

Clear consistency model

Best-effort isolation with row-level coordination and eventual index consistency.
